# H3 (Shuttle Native)

## Overview

H3 is Shuttle's native HTTP/3-over-QUIC transport with Chrome TLS fingerprinting and HMAC-based authentication — it offers the best performance and resistance to active probing among Shuttle's native protocols.

## Client Configuration

```yaml
outbounds:
  - tag: "h3-out"
    type: "h3"
    server: "server.example.com:443"
    auth_key: "your-hmac-key"
    tls:
      server_name: "server.example.com"
      insecure_skip_verify: false
    h3:
      path_prefix: "/api"
      multipath: true
```

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `tag` | string | yes | — | Unique name for this outbound |
| `type` | string | yes | — | Must be `h3` |
| `server` | string | yes | — | `host:port` of the H3 server |
| `auth_key` | string | yes | — | HMAC authentication key |
| `tls.server_name` | string | no | server host | SNI for TLS handshake |
| `tls.insecure_skip_verify` | bool | no | `false` | Skip certificate verification |
| `h3.path_prefix` | string | no | `/` | HTTP path prefix for multiplexed streams |
| `h3.multipath` | bool | no | `false` | Enable QUIC multipath for bonding multiple network paths |

## Server Configuration

```yaml
inbounds:
  - tag: "h3-in"
    type: "h3"
    listen: ":443"
    auth_key: "your-hmac-key"
    tls:
      cert_file: "/path/to/cert.pem"
      key_file: "/path/to/key.pem"
    h3:
      path_prefix: "/api"
```

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `tag` | string | yes | — | Unique name for this inbound |
| `type` | string | yes | — | Must be `h3` |
| `listen` | string | yes | — | `[addr]:port` to bind |
| `auth_key` | string | yes | — | Must match client |
| `tls.cert_file` | string | yes | — | Path to TLS certificate |
| `tls.key_file` | string | yes | — | Path to TLS private key |
| `h3.path_prefix` | string | no | `/` | Must match client |

## How It Works

1. The client opens a QUIC connection with a Chrome-mimicking TLS fingerprint.
2. An HMAC token is included in every request to authenticate the client without a login round-trip.
3. Streams are multiplexed over HTTP/3 pseudo-headers, making traffic look like normal browser-to-CDN traffic.
4. When `multipath` is enabled, the QUIC layer bonds Wi-Fi and cellular interfaces simultaneously.

## URI Format

H3 is a Shuttle-native protocol. Share server details using the Shuttle subscription format:

```
shuttle://h3?server=server.example.com:443&auth_key=KEY&sni=server.example.com#name
```

## Compatibility

H3 is Shuttle-native and has no direct equivalent in other tools. To interoperate with Clash or sing-box clients, use a compatible protocol (Shadowsocks, VLESS, Trojan) on a separate inbound.
