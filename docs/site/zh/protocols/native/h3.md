# H3（Shuttle 原生）

## 概述

H3 是 Shuttle 原生的 HTTP/3-over-QUIC 传输协议，具备 Chrome TLS 指纹模拟和基于 HMAC 的认证——在 Shuttle 原生协议中，它提供最佳性能和最强的主动探测对抗能力。

## 客户端配置

```yaml
outbounds:
  - tag: "h3-out"
    type: "h3"
    server: "server.example.com:443"
    auth_key: "your-hmac-key"
    tls:
      server_name: "server.example.com"
      insecure_skip_verify: false
    h3:
      path_prefix: "/api"
      multipath: true
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `tag` | string | 是 | — | 该出站的唯一名称 |
| `type` | string | 是 | — | 必须为 `h3` |
| `server` | string | 是 | — | H3 服务器的 `host:port` |
| `auth_key` | string | 是 | — | HMAC 认证密钥 |
| `tls.server_name` | string | 否 | 服务器主机名 | TLS 握手的 SNI |
| `tls.insecure_skip_verify` | bool | 否 | `false` | 跳过证书验证 |
| `h3.path_prefix` | string | 否 | `/` | 多路复用流的 HTTP 路径前缀 |
| `h3.multipath` | bool | 否 | `false` | 启用 QUIC 多路径，绑定多个网络路径 |

## 服务端配置

```yaml
inbounds:
  - tag: "h3-in"
    type: "h3"
    listen: ":443"
    auth_key: "your-hmac-key"
    tls:
      cert_file: "/path/to/cert.pem"
      key_file: "/path/to/key.pem"
    h3:
      path_prefix: "/api"
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `tag` | string | 是 | — | 该入站的唯一名称 |
| `type` | string | 是 | — | 必须为 `h3` |
| `listen` | string | 是 | — | 监听地址 `[addr]:port` |
| `auth_key` | string | 是 | — | 必须与客户端一致 |
| `tls.cert_file` | string | 是 | — | TLS 证书路径 |
| `tls.key_file` | string | 是 | — | TLS 私钥路径 |
| `h3.path_prefix` | string | 否 | `/` | 必须与客户端一致 |

## 工作原理

1. 客户端使用模拟 Chrome 的 TLS 指纹建立 QUIC 连接。
2. 每个请求中都包含 HMAC 令牌，无需登录往返即可完成客户端认证。
3. 流量通过 HTTP/3 伪头部进行多路复用，使流量看起来像普通的浏览器到 CDN 的请求。
4. 启用 `multipath` 时，QUIC 层会同时绑定 Wi-Fi 和蜂窝网络接口。

## URI 格式

H3 是 Shuttle 原生协议，使用 Shuttle 订阅格式分享服务器信息：

```
shuttle://h3?server=server.example.com:443&auth_key=KEY&sni=server.example.com#名称
```

## 兼容性

H3 是 Shuttle 原生协议，在其他工具中没有直接等效实现。若需与 Clash 或 sing-box 客户端互通，请在单独的入站上使用兼容协议（Shadowsocks、VLESS、Trojan）。
